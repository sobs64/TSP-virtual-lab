Traveling Salesman Problem

This experiment demonstrates the Traveling Salesman Problem (TSP), where the goal is to find the shortest possible route that visits each city exactly once and returns to the starting point. This interactive module provides a platform for visualizing and solving TSP using various algorithms, helping learners understand optimization, pathfinding, and combinatorics.

Objectives:

  • Interactive Route Visualization: Allow users to see the cities, distances, and optimal paths dynamically on a graph.
  • Algorithm Comparison: Let users choose between algorithms like brute force, greedy, or genetic to see their impact on solution quality and time.
  • Understand Optimization: Highlight how algorithmic optimizations help in reducing computation time and improving route accuracy.
